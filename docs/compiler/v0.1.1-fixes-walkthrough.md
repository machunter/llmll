# LLMLL v0.1.1 High Priority Fixes — Walkthrough

## What Was Implemented

### Fix 2 — Structured S-Expression Error Output

**Files:** `Diagnostic.hs`, `Main.hs`

Added `megaparsecToDiagnostic` in `Diagnostic.hs` that converts a raw Megaparsec `ParseErrorBundle` into a typed `Diagnostic` with source span (line/col). All parse failure paths in `Main.hs` now call `emitParseDiag`, which formats the spec-compliant output:

```
(error :phase parse :file "/tmp/bad.llmll" :line 1 :col 1
       :message "| 1 | (bad\nunexpected '('\nexpecting end of input"
       :hint "use def-logic, type, import, or check at the top level (v0.1.1 single-file model)")
```

The REPL also now uses `formatDiagnosticSExp` for parse errors. JSON output (`--json` flag) uses `formatDiagnosticJson`.

---

### Fix 1 — Missing Standard Library Functions

**File:** `Codegen.hs`

Added to the generated `lib.rs` preamble:

| Function | Notes |
|---|---|
| `string_slice(s, start, end)` | `.chars().skip().take()` — used by tictactoe parse-input |
| `string_to_int(s)` | Returns `Adt("Success", [Int(n)])` or `Adt("Error", [Text(e)])` |
| `ok(v)` | `Adt("Success", [v])` |
| `err(e)` | `Adt("Error", [e])` |
| `is_ok(r)` | Matches on `Adt("Success", _)` |
| `unwrap(r)` | Extracts from `Success`, panics on `Error` |
| `unwrap_or(r, d)` | Returns default on `Error` |
| `Sub` / `Mul` / `Div` | Mirror existing `Add` impl |

Also added `runCargoCheck` in `Main.hs` which runs `cargo check --manifest-path <out>/Cargo.toml --quiet` after codegen. The "OK Generated Rust crate" message **only appears if cargo check passes**.

---

### Fix 3 — Module/Import Top-Level Handling

**File:** `Parser.hs`, `Main.hs`

Added `parseTopLevel` as the single unified entry point (replaces the brittle two-attempt fallback). It uses `pTopLevelItem` which handles:

- A `(module Name imports body)` wrapper → **flattened** into its body statements (module name discarded per v0.1.1 single-file model)  
- Any regular statement (`def-logic`, `type`, `import`, `check`)

The `parseSrc` function in `Main.hs` was updated to use `parseTopLevel` exclusively and return `Either Diagnostic [Statement]` instead of `Either Text [Statement]`.

---

### LLMLL.md Documentation Updates

Three v0.1.1 limitation notes added:

- **§3.2** — `pair-type` is not accepted in `typed-param` position; use untyped params. Includes code example showing correct workaround.
- **§8** — Single-file model fully documented: `(module ...)` is accepted and flattened, `(import ...)` is parsed but not resolved, `wasi.*` functions are in the stdlib preamble unconditionally, capability enforcement deferred to v0.2.
- **§12 Grammar Key Rules** — Added rule 8: `pair-type` in `typed-param` is rejected in v0.1.1.
